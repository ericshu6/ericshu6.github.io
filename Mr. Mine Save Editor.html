Mr. Mine Save Editor (info)

Updated with various data for v44 (Lastly changed 16.08.2023)

I've noticed an increasing interest in the save editor. A lot of work is necessary to update it, especially after the devs went crazy changing everything. I don't really earn anything from the save editor as my website has no ads, trackers or crypto miners, so I tend to just work on the save editor when I feel like it. If you want to incentivize more frequent updates, you can Donate here

Warning: If you want to change the drill parts from the beginning of the game, you need to check "Crafted a blueprint" in the progress section. If you don't, the game will freeze whenever you open the crafting menu.

Note:There is always a possibility that some bug in the save editor might change something in an unwanted way, or might not be able to handle values that have increased beyond a known max value. Always save your unaltered export code before importing from the save editor!

Reactor Grid (sections)

The reactor grid is 9x9 with 0,0 in the top-left corner. Each square represents one row. For fuel cells, the energy field needs to be at least 0 to work. For other items, the energy field should be -1. Items can be put in parts of the grid which are not unlocked.

MrMine item info (edit)

Scientist info

Scientist level: Minutes excavated / 360 + 1.

Time until death: How long to excavate for before killing the scientist. If it's 0, the scientist won't die.

A scientist must have either an active excavation or two excavation choices. If the duration of an excavation is 0, the excavation slot is set to empty.

Trading

Trading is quite statically coded and which values are valid depends on other values. Valid trades are defined in tradingpost.js in the variable traders.

Payment subtype only matters for ore payment type and is ore id as defined in the mineral list down below.

Reward subtypes are a bit more complicated. For ore reward type, reward subtype must be a mineral in the mineral list. For chest reward type, reward subtype must be 0 for wooden chest or 1 for gold chest. For relic reward type, valid values are defined in the relic list. For other reward types, -1 seems to be the only valid value. Buffs and blueprints seem to still be unfinished and will probably be properly defined in the future.

Trade amounts should be 1 for chests, relics, blueprints and buffs, but can by any positive value for money and minerals.

Chests

Chests spawn every 30 seconds and each 10km can contain one chest, and is rolled independently. A chest will despawn after 1hr-2.4hr. Chests will overwrite other chests when spawning. This can make another chest disappear after just 30 seconds, including gold chests. If a chest will spawn, that chest has a 1 in 158 chance of spawning as a gold chest. Except for rewards, the two chest types behave the same.
